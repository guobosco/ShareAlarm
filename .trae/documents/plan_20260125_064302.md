# 实现时间选择器的时间范围限制

## 问题分析

当前的时间选择器实现没有任何时间范围限制，用户可以选择任何时间作为事件时间或提醒时间。需要添加以下限制：

1. **提醒时间限制**：
   - 提醒时间必须早于事件时间
   - 如果用户设置提醒时间晚于事件时间，需要提示错误信息

2. **时间范围限制**：
   - 事件时间必须晚于当前时间
   - 提醒时间必须晚于当前时间

## 修复方案

### 1. 修改 TimePickerDialog 组件

**添加参数**：
- `minDate`: 最小可选日期（当前时间）
- `maxDate`: 最大可选日期（对于提醒时间，这是事件时间）

**修改日期选择逻辑**：
- 禁用选择早于 `minDate` 的日期
- 对于提醒时间，禁用选择晚于 `maxDate` 的日期

**添加时间验证**：
- 在确认时间时，验证选择的时间是否在允许范围内
- 如果不在范围内，显示错误提示

### 2. 修改 CreateReminderScreen

**添加错误提示状态**：
- 添加 `alertTimeError` 状态，用于显示提醒时间错误信息

**修改时间选择逻辑**：
- 为事件时间选择器设置 `minDate` 为当前时间
- 为提醒时间选择器设置 `minDate` 为当前时间，`maxDate` 为事件时间

**添加时间验证**：
- 在创建提醒时，验证所有提醒时间是否早于事件时间
- 如果有错误，显示错误提示并阻止创建

### 3. 实现用户反馈

**添加错误提示 UI**：
- 在提醒时间输入框下方显示错误提示
- 使用红色文本显示错误信息

**添加按钮状态管理**：
- 当有错误时，禁用创建提醒按钮
- 当用户修正错误后，重新启用按钮

## 预期结果

- 用户无法选择早于当前时间的事件时间或提醒时间
- 用户无法选择晚于事件时间的提醒时间
- 当用户尝试选择无效时间时，会收到明确的错误提示
- 只有当所有时间设置都有效时，才能创建提醒

## 技术实现

- 使用 `Calendar` 类进行日期时间比较
- 使用 `mutableStateOf` 管理错误状态
- 使用条件渲染显示错误提示
- 禁用无效的时间选择选项
- 确保用户体验流畅，错误提示清晰
